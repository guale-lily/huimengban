# 登录注册数据恢复测试场景

## 测试目标
验证跨平台登录后，用户数据（头像、名字、作品等）是否正确恢复。

## 测试步骤

### 第一阶段：浏览器A操作

#### 1. 游客模式浏览
- 打开浏览器A，访问 http://localhost:5175/
- 验证初始界面显示"游客_xxx"昵称
- 浏览创意圈、创作等页面

#### 2. 注册新账号
- 点击右上角"登录"按钮
- 点击"注册"切换到注册界面
- 输入手机号：13800138001
- 输入密码：123456
- 确认密码：123456
- 选择安全问题1：你的小学名称是什么？
- 输入答案1：测试小学
- 选择安全问题2：你的第一只宠物的名字是什么？
- 输入答案2：测试宠物
- 点击"注册"按钮

#### 3. 编辑个人资料
- 注册成功后，点击右上角"编辑"按钮
- 修改昵称为：测试用户001
- 修改个人签名为：这是一个测试账号
- 上传自定义头像（如果有）
- 点击保存

#### 4. 发布作品
- 点击底部导航"创作"
- 完成IP角色设定后生成内容
- 或点击"+"发布帖子，发布1-2条作品

#### 5. 记录数据
- 截图保存当前界面
- 记录以下信息：
  - 昵称：测试用户001
  - 签名：这是一个测试账号
  - 作品数量：X条

### 第二阶段：浏览器B操作

#### 1. 清除浏览器数据（重要！）
- **这是模拟不同设备的关键步骤！**
- 打开浏览器B（Chrome/Firefox/Edge等）
- 打开开发者工具（F12）
- 切换到 Application 选项卡
- 左侧选择 Local Storage
- 删除当前站点的所有数据
- 或者：直接在浏览器A中退出登录并清除数据

#### 2. 登录账号
- 在浏览器B中打开 http://localhost:5175/
- 验证初始状态是游客模式（显示"游客_xxx"）
- 点击右上角"登录"按钮
- 输入手机号：13800138001
- 输入密码：123456
- 点击"登录"按钮

#### 3. 验证数据恢复
验证以下数据是否正确恢复：

##### 3.1 我的页面验证
- [ ] 昵称显示为"测试用户001"（不是游客）
- [ ] 个人签名显示为"这是一个测试账号"
- [ ] 头像与之前设置的一致
- [ ] 作品数量显示正确

##### 3.2 功能验证
- [ ] 点击"我的作品"能看到之前发布的作品
- [ ] 点击创意圈能看到帖子
- [ ] 编辑个人资料后保存，数据应该保留

### 第三阶段：反向验证

#### 1. 在浏览器B修改数据
- 修改昵称为：浏览器B用户
- 发布新作品

#### 2. 切换回浏览器A
- 刷新页面（不退出登录）
- 验证数据是否同步更新

## 预期结果

### 成功标志
✅ 所有测试项目都通过

### 失败表现
❌ 登录后显示"游客_xxx"而非注册的昵称
❌ 作品列表为空（之前发布的作品消失）
❌ 头像恢复为默认图标
❌ 个人签名显示默认值

## 注意事项

1. **两个浏览器必须是不同的浏览器窗口或无痕模式**
2. **浏览器B必须清除之前的localStorage数据**
3. **注册时选择的安全问题要记住，用于忘记密码测试**
4. **如果测试失败，请刷新页面重试一次**
